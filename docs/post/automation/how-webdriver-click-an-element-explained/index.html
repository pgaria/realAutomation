How Selenium WebDriver click works in background

 Posted on January 1, 0001  |  3 min (431 mots)

Selenium Webdriver provides cool interface API’s to perform many actions on the browser and simulate the user behaviours like click or enter Text in the TextBoxes. There are many good articles about the WebDriver on internet but the purpose of this article is to understand what happens in background when we use actions like click, finding an element or getting title etc. I will be using Java as a programming language in this article.

Lets have a look at one of the @Test code which is performing steps like Opening Google.com, Finding Search Button, Click SearchButton, Get Page Title and then Quit Driver instance.

  @Test
  public void clickGoogleSearchAndGetTitle() throws Exception{
    // get Google Home Page
    driver.get("https://www.google.de/");
    // find Search Button webElement
    WebElement element = driver.findElement(By.name("btnK"));
    // Click on the webElement
    element.click();
    // Get Title of the Page
    driver.getTitle();
    // Quit Driver
    driver.quit();
  }

So how WebDriver managed to perform all the user actions on browser in above @Test ? Answer is by using proper JSON Wire Protocol used in background.

Create Session

First step is to create a session with the respective driver like chromedriver or firefoxdriver and use the same session Id to communicate between client and server.

Opening the Test URL

Now after creating a successful session with driver, next step is to open URL

After every Test finished you should quit the driver, Quit call close the browser and clean the session.
